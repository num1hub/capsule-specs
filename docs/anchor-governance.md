# Anchor Governance

Anchor governance is the repository-level metadata layer that keeps high-signal files linked, extractable, and machine-inspectable.

## What anchors do

- define stable identifiers for important repository surfaces
- support deterministic extraction and validation
- keep the architecture spine visible across docs and code
- provide input to coverage and scorecard tooling

## Scope

Anchor governance is intentionally narrow. It is designed for high-signal repository surfaces rather than every helper file or every prose fragment in the tree.

## Why it matters here

For a public reference repository, anchors make governance and architecture easier to inspect without forcing contributors to reconstruct the repository map by hand.
