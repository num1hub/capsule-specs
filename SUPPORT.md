# Support

## Getting help

- Use GitHub issues for bugs, documentation gaps, and feature requests tied to this public repository.
- Use the integration-question and contract-change templates when the request is boundary-specific.
- Use pull requests for concrete fixes to docs, schemas, and examples.
- Use private maintainer contact only for security-sensitive reports.
- Read [`docs/community-health.md`](docs/community-health.md) if you need the intended public intake paths before opening an issue.
- This repository intentionally keeps support in issues and pull requests; GitHub Discussions and the repository wiki are not part of the maintained public intake surface.

## Scope of support

This repository supports the public specification and reference surface for N1Hub and CapsuleOS. It does not provide support for private operator tooling, internal vault data, or unpublished managed-cloud features.
